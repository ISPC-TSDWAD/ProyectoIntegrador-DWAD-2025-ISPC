2. Crear ramas `feature/<tarea>` desde los issues.
3. Commits (web) y PRs hacia `develop`.
4. Revisar y mergear a `develop`.
5. Al cierre del sprint, integrar a `main` y publicar release `v0.1.0-sprint0`.
